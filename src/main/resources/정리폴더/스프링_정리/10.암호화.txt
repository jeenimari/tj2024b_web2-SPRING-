[암호화]
 1.정의 : 자료를 보호 하기위해서 사람이 이해하기 어려운 문자로 변환.
 2.목적
    1.자료보안
    2.자료 무결성 유지
    등등
 3.사용처
    1.HTTPs - 메세지(HTTP)암
    2.계정 비밀번호 - 암호화
    3.금융 거래 -암호화 등등
    등등
 4.용어
    1.암호화 : 자료를 변환하는 과정 ( 사람이 이해하기 어려운 문자로)
    2.복호화 : 암호화된 자료를 원래 상태로 변환하는 과정
    3.해시 함수 : 데이터를 고정된 길이로 변환하는 함수
    4.솔트 : 암호화할대 사용되는 랜덤값
    5.레인보우 테이블 : 미리 계산된 해시값을 이용하여 복호화를 찾는 기법
    6.단방향 암호화 : 원래 상태로 변환할 수 없는 암호화 방식(해시)
    7.양방향 암호화 : 원래 상태로 변환할수 있는 암호화 방식

 5.실무 암호화 알고리즘
  1. 비밀번호 : bcrypt 알고리즘 등
  2. 파일 검증 : SHA- 256 알고리즘 등
  3. HTTP : TLS 알고리즘 등
  4. 전자 서명 알고리즘 등

6. 비크립트(Bcrypt)
  1.정의 : 해시 함수를 이용하여 주로 비밀번호를 암호화 할대 사용됨
  2.특징 :
    1.솔트(salt) : 같은 비밀번호를 랜덤(salt)값 으로 서로 다른 암호값을 만들기 가능
        (자료)abc (salt)1234 -> a154b8734~
        (자료)abc (salt)4567 -> a354q5431~
    2.반복연산 적용 : 해독하는 속도 늦춤
    3.원본 복구 불가능(단방향 암호화 ) :(로그인) 비밀번호 검증은 자료비교가 아닌 해시값 비교함.

    4.비크립트해시 결과?
